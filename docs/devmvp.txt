Day 1 — Environment setup. Created Python venv, installed FastAPI, uvicorn, httpx, PyJWT, passlib, ultralytics, opencv-python. Scaffolded backend/main.py with health check. Connected to Supabase via backend/db.py (httpx wrapper for REST API). Created backend/.env with Supabase URL, service key, JWT secret. Tested select/insert against Supabase.
Day 2 — JWT authentication. Built backend/auth.py with POST /auth/login — accepts username + password, hashes with bcrypt, returns JWT token (60min expiry). Created get_current_user dependency for protected routes. users table stores hashed passwords only. Frontend login.html + auth-guard.js redirects unauthenticated users.
Day 3 — YOLOv8 model integration. Built backend/detector.py loading models/best.pt. 14 classes defined: 5 violations (NO-Hardhat, NO-Gloves, NO-Goggles, NO-Mask, NO-Safety Vest), 1 fall (Fall-Detected), 7 safe items (Hardhat, Gloves, etc.). detect() runs inference, annotate() draws bounding boxes (red for violations, green for safe).
Day 4 — Detection endpoint + dashboard shell. POST /detect accepts uploaded frame, runs YOLO, returns annotated image + detections. Frontend index.html has the video feed player with HUD overlay (live indicator, timestamp). dashboard.js uses getUserMedia to capture browser webcam frames every 1.5s and sends to /detect.
Day 5 — Event storage. POST /events creates a detection_events row (user_id, event_type, snapshot) plus individual detections rows for each detected object. 30-second dedup cooldown prevents duplicate events. GET /events returns paginated events with label (first detection's class name). Snapshot stored as base64 JPEG.
Day 6 — Alert system. When a violation or fall is detected, an alerts row is auto-created (linked to the event). PATCH /alerts/{id}/ack marks as acknowledged. Dashboard has live alerts ticker with acknowledge buttons. Floating toast notification + alarm sound play on new violations.
Day 7 — Violation log page. logs.html with filterable table (event type, date range), column sorting, pagination. logs.js fetches events via fetchViolations(), renders rows with timestamp, type badge, confidence, status, actions. Snapshot viewer button shows annotated image in fullscreen overlay. CSV export.
Day 8 — Reports page. reports.html with 2 charts: violation type distribution (doughnut) and daily compliance trend (line). fetchReportingData() aggregates events by type and day-of-week. Summary cards show total violations, top violation type, total events.
Day 9 — History page. history.html with date picker. history.js fetches events filtered by selected date. Summary cards show total events, violations, falls, compliant counts for that day. Event table with type, confidence, status, acknowledge button. CSV export.
Day 10 — Camera stream manager. Built backend/camera_manager.py — threaded RTSP reader per camera. CameraStream class opens cv2.VideoCapture in a background thread, stores latest frame. start_camera(), stop_camera(), get_frame() functions with thread-safe locking.
Day 11 — Camera CRUD. Added 7 camera endpoints: list, create, update, delete, start stream, stop stream, active status. settings.html + settings.js with camera management table, add/edit modal, start/stop buttons per camera. /detect accepts ?camera_id=X to read from RTSP buffer instead of uploaded file.
Day 12 — Enhanced reports. Added 3 more charts: events by camera (horizontal bar), hourly event distribution (bar), 30-day compliance trend (line). fetchReportingData() now computes hourly and daily aggregations. Reports grid layout adapts to screen width.
Day 13 — Camera integration across pages. Dashboard gets camera dropdown (below video, next to Start button) — switch between browser webcam and IP cameras. Violation log + history get Camera column showing camera name. Events store camera_id + camera_name. Nav sidebar gets Cameras link on all pages.
Day 14 — Bug fixes. Fixed label logic — now prefers violation class names (NO-*) over Person for event display. Added Chromium HTTPS warning when getUserMedia unavailable. Fixed confidence from hardcoded 0.92 to actual model confidence. Cleaned up history.js counting logic. Fixed history prev/next/today buttons.
Day 15 — Documentation & cleanup. Created MVP.md with build log. Cleaned README.md — removed references to deleted files (stream.py, DBXSTRUCTURE.TXT, pytest.ini), added frontend directory tree, fixed tech stack (Chart.js not Plotly), removed unbuilt features (RBAC, email/SMS, sites). Added cache-busting query params across all pages. Removed dead files.
